## 1. 現状分析

### 1.1 現在の技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js 16.1.6 (React 19) |
| スタイリング | Tailwind CSS 4 |
| 多言語対応 | next-intl（日本語・英語） |
| フォント | Noto Sans JP / Noto Serif JP |

### 1.2 現在の良い点

- ✅ セマンティックなHTML構造（header, nav, main, footer）
- ✅ スキップリンクの実装
- ✅ ARIA属性の使用（aria-label, aria-pressed, aria-expanded）
- ✅ 適切な見出し階層（h1 → h2 → h3）
- ✅ 行間 1.8 で読みやすい本文
- ✅ レスポンシブデザイン対応

### 1.3 改善が必要な点

| 項目 | 現状 | 課題 |
|------|------|------|
| 本文フォントサイズ | 16px | 50代以上には小さい可能性 |
| ナビゲーション文字 | 14-16px | 視認性に課題 |
| ボタンサイズ | 標準サイズ | タップ領域が狭い |
| アクセントカラー | #6B4C9A（コントラスト比 4:1） | WCAG AA ギリギリ |
| リンクの識別 | 下線なし | リンクと認識しにくい |
| 言語切替ボタン | 小さい | 操作しにくい |

## 2. 改善提案

### 2.1 フォントサイズの拡大

**目的**: 老眼や視力低下があっても快適に読める文字サイズを確保

| 要素 | 現在 | 提案 | 根拠 |
|------|------|------|------|
| 本文テキスト | 16px | **18px** | JIS X 8341-3 推奨 |
| ナビゲーション | 14-16px | **16-18px** | 操作要素は大きめに |
| ボタンテキスト | 14-16px | **18px** | タップ対象の視認性向上 |
| 小さい文字（日付等） | 12-14px | **16px以上** | 最小フォントサイズ確保 |
| フッター | 12-14px | **14-16px** | 補足情報も読みやすく |

### 2.2 コントラストの強化

**目的**: WCAG 2.1 AAA基準（コントラスト比 7:1）を目指す

| 要素 | 現在の色 | 提案色 | コントラスト比 |
|------|----------|--------|----------------|
| アクセントカラー | #6B4C9A (4:1) | **#5A3D8A** | 5.5:1 以上 |
| グレーテキスト | #525252 (7:1) | **#404040** | 9:1 以上 |
| リンク色 | グレー系 | **#2563EB**（青） | 4.5:1 以上 + 下線 |
| プレースホルダー | #9CA3AF | **#6B7280** | 4.5:1 以上 |

### 2.3 タップ/クリックターゲットの拡大

**目的**: 指先の精度が低下しても確実に操作できるサイズを確保

| 要素 | 現在 | 提案 | 基準 |
|------|------|------|------|
| ボタン | 約36px高 | **最小48px × 48px** | WCAG 2.5.5 推奨 |
| ナビゲーションリンク | パディング小 | **上下16px、左右24px** | タップ領域拡大 |
| 言語切替ボタン | 小さい | **44px × 44px以上** | モバイル操作性 |
| フォーム入力欄 | 標準 | **高さ48px以上** | 入力しやすさ |

### 2.4 視覚的な改善

#### リンクの識別性向上

```
【現在】
リンクテキストがグレーで下線なし → リンクと認識しにくい

【提案】
- テキストリンクには常時下線を表示
- ホバー時は下線を太くする or 背景色を追加
- 訪問済みリンクの色を変更
```

#### 字間・余白の調整

| 項目 | 現在 | 提案 |
|------|------|------|
| 字間（letter-spacing） | 標準 | **0.05em** |
| 段落間の余白 | 標準 | **現在の1.2倍** |
| セクション間の余白 | 標準 | **現在の1.2倍** |

### 2.5 ナビゲーションの改善

| 改善項目 | 内容 |
|----------|------|
| 現在地の明示 | アクティブなページを背景色やボーダーで強調 |
| パンくずリストの強化 | より大きな文字サイズ、明確な区切り記号 |
| ホームへの導線 | ロゴに加えて「ホーム」テキストリンクを追加 |
| メニューの間隔 | 各項目間の余白を広げる |

### 2.6 フォーム（お問い合わせ）の改善

| 項目 | 提案内容 |
|------|----------|
| ラベル | 入力欄の上に常時表示（プレースホルダーのみに頼らない） |
| 入力欄 | 高さ48px以上、ボーダー2px |
| エラーメッセージ | 赤色 + アイコン + 大きな文字 |
| 送信ボタン | 目立つ色、大きなサイズ（最低48px高） |
| 入力補助 | 任意/必須の明示、入力例の表示 |

### 2.7 新機能の追加提案

#### 2.7.1 文字サイズ変更機能

ヘッダーに「小・中・大」の文字サイズ切替ボタンを設置

```
[A-] [A] [A+]  または  [小] [中] [大]
```

**機能仕様**:
- デフォルト: 中（18px）
- 小: 16px
- 大: 20px
- 設定はローカルストレージに保存

#### 2.7.2 印刷用スタイル

重要な情報（礼拝時間、アクセス情報等）を印刷しやすいレイアウトで提供

#### 2.7.3 ハイコントラストモード（オプション）

視覚障害のある方向けに、より高いコントラストのカラースキームを提供

## 4. 技術的な実装方法

### 4.1 globals.css の変更例

```css
/* ベースフォントサイズの拡大 */
html {
  font-size: 18px; /* 16px → 18px */
}

/* リンクに下線を追加 */
a:not(.btn-primary):not(.btn-secondary) {
  text-decoration: underline;
  text-underline-offset: 2px;
}

a:hover {
  text-decoration-thickness: 2px;
}

/* ボタンの最小サイズ */
.btn-primary,
.btn-secondary {
  min-height: 48px;
  min-width: 48px;
  padding: 12px 24px;
}

/* フォーム入力欄 */
input[type="text"],
input[type="email"],
textarea {
  min-height: 48px;
  font-size: 18px;
}

/* 字間の調整 */
body {
  letter-spacing: 0.05em;
}
```

### 4.2 Tailwind 設定の変更例

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      fontSize: {
        base: '18px',  // デフォルトを18pxに
      },
      colors: {
        accent: '#5A3D8A',  // より濃い紫
        'gray-600': '#404040',  // より濃いグレー
      },
    },
  },
}
```

### 4.3 文字サイズ変更コンポーネント例

```tsx
// components/FontSizeToggle.tsx
'use client';

import { useState, useEffect } from 'react';

export function FontSizeToggle() {
  const [size, setSize] = useState<'small' | 'medium' | 'large'>('medium');

  useEffect(() => {
    const saved = localStorage.getItem('fontSize');
    if (saved) setSize(saved as typeof size);
  }, []);

  useEffect(() => {
    document.documentElement.style.fontSize =
      size === 'small' ? '16px' : size === 'large' ? '20px' : '18px';
    localStorage.setItem('fontSize', size);
  }, [size]);

  return (
    <div className="flex gap-2" role="group" aria-label="文字サイズ">
      <button onClick={() => setSize('small')} aria-pressed={size === 'small'}>小</button>
      <button onClick={() => setSize('medium')} aria-pressed={size === 'medium'}>中</button>
      <button onClick={() => setSize('large')} aria-pressed={size === 'large'}>大</button>
    </div>
  );
}
```

## 5. 期待される効果

### 5.1 ユーザー体験の向上

| 改善項目 | 期待される効果 |
|----------|----------------|
| フォントサイズ拡大 | 老眼の方でも眼鏡なしで読める |
| コントラスト強化 | 明るい環境でも文字が読みやすい |
| タップ領域拡大 | スマートフォンでの誤タップ減少 |
| リンク下線追加 | クリック可能な要素が一目でわかる |
| 文字サイズ変更機能 | 個人の視力に合わせた調整が可能 |

### 5.2 アクセシビリティ基準への適合

| 基準 | 現状 | 改善後 |
|------|------|--------|
| WCAG 2.1 コントラスト | AA（一部） | AA（全体）〜 AAA |
| JIS X 8341-3 | レベルA | レベルAA |

### 5.3 想定される利用シーン

1. **礼拝時間の確認** - 大きな文字で読みやすく
2. **お問い合わせフォーム入力** - 入力欄が大きく操作しやすい
3. **お知らせの閲覧** - 記事本文が読みやすい
4. **スマートフォンでのアクセス** - タップしやすいボタン
